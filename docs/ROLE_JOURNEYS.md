# ROLE_JOURNEYS.md

## 1. 产品负责人 / PM

关注页面：Page 1、Page 5、Page 6、Page 7。

决策动作：定义 Run 边界，确认推荐候选，决定是否生成 PRD 草案，决定是否发起飞书评审会议和验证任务。

典型判断：跨平台 Dock 兼容预检助手可带 Warning 进入 PRD 草案，但必须先锁定支持拓扑、准确率责任和版本维护边界。

## 2. 用户研究员

关注页面：Page 2、Page 3、Page 7。

决策动作：确认样本质量，修正痛点标签，补充用户原声，提出用户访谈任务。

典型判断：线缆混乱和功率分配不透明是高频痛点，但需内部售后数据复核。

## 3. 竞品 / 商业分析负责人

关注页面：Page 2、Page 4、Page 5。

决策动作：补充竞品 SKU，确认机会空白，标记产品重叠风险，参与差异度评分。

典型判断：兼容预检必须证明比现有 FAQ 或静态兼容列表更能降低用户试错成本。

## 4. 工业设计师

关注页面：Page 3、Page 5、Page 6。

决策动作：确认场景路径，输出形态方向，提出概念草图任务。

典型判断：当前候选不是新增 Dock 硬件形态；工业设计重点是清晰表达设备拓扑、风险和可执行修复建议。

## 5. 硬件 / 制造负责人

关注页面：Page 4、Page 5、Page 6。

决策动作：评估技术复杂度，标记制造风险，压缩 MVP 功能。

典型判断：P0 不新增硬件，先验证设备识别、规则匹配、风险解释和修复建议的可实现性。

## 6. 市场 / 增长负责人

关注页面：Page 3、Page 5、Page 6。

决策动作：判断目标用户、卖点和付费意愿验证方式。

典型判断：多设备办公场景清晰，但仍需验证预检任务价值、信任形成和付费/留存意愿。
