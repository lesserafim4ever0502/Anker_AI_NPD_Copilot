# FEISHU_COLLABORATION.md

## 1. Feishu Positioning

飞书不是 Anker AI-NPD Copilot 的附属导出工具，而是其企业协同底座：多维表格承载结构化证据，文档承载 PRD 与评审报告，会议与妙记承接二次确认，任务承接 MVP 验证计划，知识库沉淀可复用的 NPD 组织资产。

## 2. Four-layer Model

### Layer 1: 输入数据承载

- 安克产品矩阵
- 竞品产品矩阵
- 用户反馈样本
- 痛点标签体系
- 候选 NP 评分表
- 数据缺口表

飞书对象：多维表格、文档、知识库。

### Layer 2: AI 协同分析

- 测评资料摘要
- 用户痛点提炼
- 竞品对比摘要
- 产品评审会议纪要
- PRD 文档初稿生成
- 历史知识问答

飞书对象：飞书 AI、飞书妙记、飞书文档、知识库问答。

### Layer 3: 输出产物沉淀

- 样例 NP PRD 摘要
- Stage-Gate 决策记录
- 多 Agent 产品评审报告
- 风险与待确认事项
- MVP 验证任务清单
- 二次评审会议议程

飞书对象：文档、多维表格、任务、会议。

### Layer 4: 知识复用

- 用户痛点知识库
- 竞品情报知识库
- 历史候选 NP 评审库
- 风险案例库
- PRD 模板库
- NPD 方法论库

飞书对象：知识库、文档目录、模板。

## 3. Demo Sync Strategy

报名阶段：Web Demo + 本地 JSON + 飞书工作区样例 / Mock 同步。

入围后：通过飞书 CLI 半自动同步 Markdown PRD、评分表和验证任务。

后续版本：通过飞书开放平台 API 接入多维表格、文档、任务和知识库。

## 4. Required Demo Artifacts

至少展示：

- 飞书多维表格：产品与竞品矩阵。
- 飞书多维表格：用户反馈与痛点标签。
- 飞书文档：PRD 摘要草案。
- 飞书文档：多 Agent 评审报告。
- 飞书任务：MVP 验证任务清单。
- 飞书会议：二次评审会议议程。
- 飞书知识库：NPD Run 案例归档。

## 5. Key Principle

系统输出不应只导出一个文档，而应拆分为文档、表格、任务、会议和知识库节点，支持后续真实推进。
