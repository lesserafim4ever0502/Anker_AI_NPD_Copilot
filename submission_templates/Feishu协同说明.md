# 飞书协同说明

Anker AI-NPD Copilot 与飞书采用“数据承载—AI 协同—产物沉淀—知识复用”的四层协同模式。

输入端，系统将安克产品矩阵、竞品矩阵、用户反馈样本、痛点标签体系和候选 NP 评分沉淀在飞书多维表格中，将测评文章、调研笔记、Agent 角色说明和方法论文档沉淀在飞书文档与知识库中，并借助飞书 AI 对长文本资料进行摘要、分类与结构化。

当前已通过飞书 CLI 创建真实数据采集 Base、五张原始采集表、后续清洗/评审表，以及数据采集填写说明和 Proposal PRD 文档。该协作工作台独立于网页运行时。

输出端，系统将样例 NP PRD 摘要、多 Agent 评审报告、Stage-Gate 决策记录和 MVP 验证计划输出为结构化 Markdown / JSON，并通过人工复制、飞书 CLI 或后续开放平台 API 写回飞书文档、多维表格、任务和知识库。

网页 Demo 中的 Feishu Handoff 是 Proposal-stage Mock，不读取 CLI 登录状态、不暴露 token，也不声称已经自动创建任务、会议或知识库。

沉淀端，飞书知识库保存每一次新品定义的证据链、决策过程、否决理由、PRD 模板和验证结果，使单次创新项目转化为可复用的组织知识资产。
