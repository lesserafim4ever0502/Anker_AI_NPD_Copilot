# EVIDENCE_TO_DECISION.md

## 1. Core Idea

Evidence-to-Decision 是 Anker AI-NPD Copilot 的核心方法论。它要求每个新品建议都能追溯到数据来源、用户痛点、竞品边界、门禁评审和多 Agent 意见。

## 2. Flow

```text
Raw Evidence
  → Structured Fields
  → Pain Tags
  → Design Signals
  → Opportunity Gaps
  → Candidate NP
  → Stage-Gate Review
  → Multi-Agent Review
  → PRD Draft
  → Feishu Archive
```

## 3. Why It Matters

普通 AI 创意工具的问题是：输出很快，但难以解释为什么做、为什么不做、风险在哪里、后续谁负责。

Evidence-to-Decision 的目标是让企业产品团队能复盘：

- 为什么生成这些候选 NP？
- 为什么推荐 A 而不是 B 或 C？
- 哪些风险还没解决？
- 哪些结论依赖公开样本，哪些需要内部数据复核？
- 下一步任务和会议如何进入飞书？

## 4. Demo Requirement

每个核心页面都必须显示证据、状态或下一步动作，避免成为孤立展示页。
