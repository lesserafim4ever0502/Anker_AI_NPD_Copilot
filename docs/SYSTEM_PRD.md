# SYSTEM_PRD.md

## 1. Product Overview

Anker AI-NPD Copilot 是面向安克新品定义团队的 AI 原生产品研发决策工作台。系统支持多项目状态管理，并通过一次 NPD Run 展示从公开证据池到 PRD 草案、再到飞书协同沉淀的完整流程。

## 2. Problem Statement

传统新品定义容易受到以下问题影响：

- 用户反馈分散，难以转化为产品语言。
- 竞品参数很多，但缺少与用户痛点的交叉判断。
- 新品创意容易拍脑袋或陷入硬件堆料。
- 评审过程中的反对意见和否决理由没有结构化沉淀。
- PRD 草案、会议、任务和知识复用之间割裂。

## 3. Target Users

- 产品负责人 / PM
- 用户研究员
- 竞品 / 商业分析负责人
- 工业设计师
- 硬件 / 制造负责人
- 市场 / 增长负责人
- 企业评审专家

## 4. Core Value Proposition

系统帮助安克新品团队完成：

1. 多项目 NPD 状态追踪。
2. 公开小样本证据池构建。
3. 用户痛点结构化。
4. 竞品机会识别。
5. 候选 NP 生成与筛选。
6. Stage-Gate 与多 Agent 产品委员会评审。
7. PRD 摘要草案输出。
8. 飞书文档、表格、任务、会议和知识库沉淀。

## 5. Product Scope

### P0

- 多项目工作台。
- 主 NPD Run 的 7 页流程。
- 本地 JSON 驱动页面。
- Mock 飞书同步。
- Decision Panel、Run Status Bar、Decision Ledger。

### P1

- 筛选、切换、详情展开。
- Mock 任务与会议创建。
- 状态更新提示。

### P2

- Markdown 导出。
- 真实飞书链接。
- CLI 同步演示。

## 6. Functional Requirements by Page

### Page 1: 项目工作台

用户可以查看 3 个 NPD 项目，并进入主项目。页面显示项目状态、推荐候选、待确认数量、下一步动作和 Owner。

### Page 2: 证据池概览

展示安克产品 SKU、竞品 SKU、用户反馈、有效痛点样本、测评观点和可引用用户原声。展示数据来源、证据卡片、证据链流程和数据缺口。

### Page 3: 用户痛点雷达

展示痛点排行榜、痛点簇、用户原声、设计信号和飞书用户洞察层。

### Page 4: 竞品机会矩阵

展示品牌覆盖矩阵、产品能力表、机会空白、候选 NP 映射和产品重叠风险。

### Page 5: 决策评审

展示 3 个候选 NP、Stage-Gate 矩阵、Kill Criteria、多 Agent 评审、推荐结果和待确认事项。

### Page 6: PRD 输出

展示样例 NP PRD 摘要草案、证据链、MVP 功能、非目标、风险、验证计划、飞书导出项。

### Page 7: 飞书沉淀

展示飞书四层架构、飞书产物卡片、二次评审会议、验证任务、Decision Ledger、知识库目录和自动化路线。

## 7. Non-functional Requirements

- Demo 必须稳定、可离线展示。
- 页面文案必须清楚标注公开小样本和 Mock 同步边界。
- 页面信息应适合人类阅读和 Agent 摘要。
- 不依赖网络接口完成核心展示。
- 所有关键结论应有证据链、状态或待确认项。

## 8. Success Metrics for Submission

- 人类评委能在 3–5 分钟内理解流程。
- AI 辅助审阅能准确摘要项目不是创意生成器，而是决策系统。
- Demo 能展示飞书协同不是硬塞。
- 方案体现企业 NPD 真实工作流：二次确认、上会、任务、归档。
